What is Agentwise?

Agentwise revolutionizes software development by orchestrating multiple specialized AI agents working in parallel. Built as an extension for Claude Code, it enables teams of AI agents to collaborate on complex projects with unprecedented efficiency.

Installation

Quick Install (Recommended)

macOS / Linux

curl -fsSL https://raw.githubusercontent.com/VibeCodingWithPhil/agentwise/main/installers/install.sh | bash

# Download and run installer
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/VibeCodingWithPhil/agentwise/main/installers/install.ps1" -OutFile "install.ps1"

cd agentwise

# 2. Install dependencies

cd ../..

# 5. Start Claude Code with required flag
claude --dangerously-skip-permissions

üèóÔ∏è System Architecture

Overview

Agentwise employs a sophisticated multi-agent orchestration architecture that enables parallel execution, intelligent task distribution, and seamless integration with Claude Code.

Core Components

1. Orchestrator

The central brain that coordinates all agent activities:

  • Dynamic Agent Manager - Discovers and loads agents automatically
  • Task Distributor - Intelligently assigns tasks to appropriate agents
  • Phase Controller - Manages multi-phase project execution
  • Token Optimizer - Achieves 30-40% token reduction

2. Agent System

Specialized agents for different aspects of development:

  • Frontend Specialist - UI/UX implementation
  • Backend Specialist - API and server logic
  • Database Specialist - Data modeling and optimization
  • DevOps Specialist - CI/CD and infrastructure
  • Testing Specialist - Quality assurance
  • Security Specialist - Security implementation
  • Designer Specialist - UI/UX design
  • Code Review Specialist - Code quality

3. Integration Layer

  • MCP Integration - 26+ MCP servers for enhanced capabilities
  • Model Router - Smart routing between Claude, Ollama, and local models
  • Context Manager - Persistent project context

Data Flow

  1. User issues command in Claude Code
  2. Orchestrator analyzes requirements
  3. Task Distributor creates agent-specific tasks
  4. Agents execute in parallel with shared context
  5. Phase Controller synchronizes progress
  6. Monitor dashboard provides real-time updates

Token Optimization Strategy

Our optimization achieves 30-40% token reduction through:

  • Context sharing between agents
  • Incremental updates instead of full context
  • Agent pooling (max 3 simultaneous)
  • Smart context windowing
  • Response caching

‚ö° Dynamic Agents

How It Works

The Dynamic Agent System enables Agentwise to automatically discover, load, and manage agents without any hardcoded references.

Key Components

  • DynamicAgentManager - Handles agent discovery and loading
  • DynamicTaskDistributor - Intelligently distributes tasks
  • AgentSelector - Chooses appropriate agents for projects

Agent Discovery Process

  1. System scans .claude/agents/ folder
  2. Parses each .md file for agent metadata
  3. Registers agents with the orchestrator
  4. Updates agent pool every 5 seconds

Smart Task Distribution

Tasks are distributed based on:

  • Agent expertise and capabilities
  • Current workload and availability
  • Project requirements and technology stack
  • Inter-agent dependencies

Troubleshooting

Connection Issues

Problem: Cannot connect to Figma MCP Server
Solutions:
  • Ensure Figma desktop app is running
  • Enable Dev Mode MCP Server in Preferences
  • Check firewall settings for port 3845
  • Restart Figma desktop app

Generation Failures

Problem: Code generation fails
Solutions:
  • Select a valid frame/component in Figma
  • Ensure selection has proper structure
  • Check component naming (avoid special characters)
  • Verify you have proper Figma plan access

API Reference

// Initialize connection
await client.initialize();

// Generate code from selection
const code = await client.getCode({
  framework: 'react',
  includeStyles: true,
  useCodeConnect: true
});

// Get design variables
const variables = await client.getVariableDefinitions();

// Export design tokens
await client.exportDesignTokens('./tokens.json');

// Capture screenshot
await client.getImage('./screenshot.png');

// Create design rules
await client.createDesignSystemRules('./design-system');

Security Considerations

  • Local Server Only: MCP Server runs locally (127.0.0.1)
  • No External Access: Cannot be accessed from outside
  • Secure Communication: All data stays local
  • No Credentials Stored: Uses Figma desktop session

Limitations

  • Beta Feature: Some functionality may be unstable
  • Desktop Only: Requires Figma desktop app
  • Plan Requirements: Needs paid Figma plan with Dev Mode
  • Selection Based: Works on current selection only

üîß Troubleshooting

Common Issues

Claude Code Permission Error

Problem

Commands fail with permission errors

Solution: Start Claude Code with required flag:

claude --dangerously-skip-permissions

Monitor Won't Open

Problem

Monitor dashboard doesn't open in browser

Solutions:

  • Check if port 3001 is available
  • Manually open: http://localhost:3001
  • Restart monitor: /monitor

Agent Not Found

Problem

Custom agent not being discovered

Solutions:

  • Check file location: .claude/agents/
  • Verify file extension: .md
  • Check markdown format is correct
  • Wait 5 seconds for discovery

Token Limit Exceeded

Problem

API rate limits or token limits hit

Solutions:

  • Enable local models: /setup-ollama
  • Configure routing: /configure-routing optimize
  • Reduce parallel agents in settings
